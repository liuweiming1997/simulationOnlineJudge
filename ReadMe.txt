------模拟OJ评判器------
具体思路：
用户需要提供三段代码，第一段是自己的代码，第二段是标程，第三段是生成对应这一道题的数据代码。程序是这样的，
用三个TextArea来接收三段代码，然后分别用GCC编译，生成三个.exe文件，首先运行生成数据的代码，生成数据，再调用另外两个代码，生成对应的答案，比较即可。

细节：
1、需要为每段代码都插上freopen()语句，使得每段代码都能重定向文件输入和输出。做法是判断是否找到了main，找到main后，在第一个'{'后插上这些语句。
2、这是一个还没修复的bug，编译时的速度达不到代码的运行速度，比如有两句代码，第一句是要去编译另外一段代码，生成一个.exe文件，第二句就要运行这个.exe文件，那么问题来了，第二句运行的时候说XXX.exe不存在，意思就是还没编译完。这个要怎么解决？我只是强行等待1.5s就好了。
3、同样的道理，运行标程代码和我的代码，去生成答案的时候，也是很慢，这里又要等待1.5s才去比较两个输出文件是否相同。
我觉得这样的用户体验很差，但是想不到有什么方法改进，不知道那些OJ是怎么写的。
